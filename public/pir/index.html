StarScope

StarScope provide a convenient way t' run th' scRNA-seq workflow which uses starsolo (https://github.com/alexdobin/STAR/blob/master/docs/STARsolo.md) an' Seurat package (https://satijalab.org/seurat/) as th' core modules t' process scRNA-seq data an' generate a concise HTML report. Th' workflow was implemented by nextflow.

Quick Start wit' ThunderBio Example Data

ThunderBio will provide human an' mouse pre-built reference an' two sets example data upon request. Th' demo_data fold contains two directories, human an' mouse. In each directory, user could invoke th' pipeline by runn'n command below. Please note th' reference index has t' be placed 'n th' same parent folder o' th' demo_data.

## Human Data Test'n wit' conda or docker
cd demo_data/human
starscope run --input sampleList.csv --config thunderbio_human_conda.config
starscope run --input sampleList.csv --config thunderbio_human_docker.config
## Mouse Data Test'n wit' conda or docekr
cd demo_data/mouse

Installat'n

Dependencies

  • Java 11 or higher
  • Nextflow
  • Conda/Miniconda
  • or Docker Engine

StarScope will automatically check dependencies, install nextflow an' create conda environment if conda be selected as runn'n environment. But user will have t' install Java an' Miniconda manually. It be suggested t' install mamba via conda install -n base -c conda-forge mamba t' speedup environment initiat'n process. By default, th' nextflow binary will be downloaded 'n th' work'n directory, user could move it t' $PATH. Alternatively, user could use docker container as runn'n environment. But docker has t' be installed on th' system:

Java

StarScope was tested wit' both openJDK an' oracle Java SE version.

## download link may vary depend'n on java version
wget -c https://download.java.net/java/GA/jdk17.0.2/dfd4a8d0985749f896bed50d7138ee7f/8/GPL/openjdk-17.0.2_linux-x64_bin.tar.gz
## Java SE version
## wget -c https://download.oracle.com/java/17/latest/jdk-17_linux-x64_bin.tar.gz
tar xvzf openjdk-17.0.2_linux-x64_bin.tar.gz
## set environment vari'ble
export JAVA_HOME="$(pwd)/jdk-17.0.2"
export PATH="$(pwd)/jdk-17.0.2/bin:$PATH"
## it be suggested t' add export cmd above t' yer .bashrc

Conda

Please use th' correspond'n installat'n script fer yer processor architecture.

wget -c https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh

starscope will create conda environment automatically when invok'n run subcommand first time wit' th' --conda opt'n. User could create th' environment manually before runn'n.

conda env create -f starscope/scRNA-seq/scRNAseq_env.yml

Docker

curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

Then login out an' login again fer th' changes t' take effect.

Prepare STAR reference

User could use their own genome FASTA an' GTF file t' create STAR reference by invok'n starscope mkref command. Example below used Zebrafish genome files as input.

## Zebrafish genome files be from Ensembl release 107
## https://uswest.ensembl.org/Danio_rerio/Info/Index
## Download genome FASTA file
wget -c http://ftp.ensembl.org/pub/release-107/fasta/danio_rerio/dna/Danio_rerio.GRCz11.dna.primary_assembly.fa.gz
## Download GTF file
wget -c http://ftp.ensembl.org/pub/release-107/gtf/danio_rerio/Danio_rerio.GRCz11.107.gtf.gz

mkref command:

starscope mkref --docker \
                --genomeFasta Danio_rerio.GRCz11.dna.primary_assembly.fa \
                --gtf Danio_rerio.GRCz11.107.gtf \
                --refoutDir Danio_rerio.GRCz11.107_STAR

We included a bash script fer user t' generate 10x compat'ble reference. 10x has removed some genes from th' standard genecode annotat'n. Please refer t' their website fer detail process procedures.

T' generate human reference wit' conda, use command below:

prepare_10x_compatible_reference.sh human --cpus 8 --mem 32.GB -bg

user could use prepare_10x_compatible_reference.sh -h t' get th' full help:

prepare_10x_compatible_reference.sh will help ye t' generate
10x cellranger compat'ble STAR reference set. Th' detail prepar'n
procedures could be referred from 10x's documentat'n website:
https://support.10xgenomics.com/single-cell-gene-expression/software/release-notes/build#header

Usage:
prepare_10x_compatible_reference.sh <human|mouse|hm|all> [starscope_options]

prepare_10x_compatible_reference.sh human    generate human GRh38 reference
prepare_10x_compatible_reference.sh mouse    generate mouse mm10 reference
prepare_10x_compatible_reference.sh hm       combine GRh38 an' mm10 t'
                                             generate reference fer hybrid
                                             sample analysis
prepare_10x_compatible_reference.sh all      generate all three reference
                                             datasets mentioned above

starscope_opt'ns:
--executor        Define executor o' nextflow (local), see:
                  https://www.nextflow.io/docs/latest/executor.html
--cpus            CPUs t' use fer all processes (8)
--mem             Memory t' use fer all processes, please note
                  th' special format (16.GB)
--noDepCheck      Do not check Java an' nextflow before
                  runn'n (false)
-bg               Runn'n th' pipeline 'n background (false)

example:

Usage

Please note th' whitelist files have t' be decompressed before pass'n t' th' program. T' run ThunderBio dataset, decompress wit' gunzip whitelist/V2_barcode_seq_210407_concat.txt.gz an' use th' /path/to/V2_barcode_seq_210407_concat.txt as th' whitelist file 'n starscope run command.

Please also note --mem opt'n o' run command needs a special format: e.g. 32.GB.

Please note user could use -bg t' run th' command 'n th' background, an' --executor slurm will submit th' jobs t' HPC slurm job scheduler.

Example sample list

Th' workflow takes a csv format sample list as input, which be th' sampleList.csv 'n th' command above. Th' sample list file only contains three column: sample, fastq_1, fastq_2. User only needs t' specify sample name, path o' read1 an' read2. Th' program will assume that read1 file be th' one contain'n barcode an' UMI informat'n. Both absolute an' relative path be supported. If th' library has multiple runs (i.e. multiple files o' R1 an' R2), user could indicate th' path o' those files wit' identical sample name, an' they will be cat into one 'n th' analysis.

sample,fastq_1,fastq_2
sampleID,read1.fq.gz,/absolute/path/to/read2.fq.gz

For GEX+VDJ analysis, sampleList.csv will require an additional column feature_types.

sample,fastq_1,fastq_2,feature_types
vdj_sample1,gex_R1.fq.gz,gex_R2.fq.gz,GEX
vdj_sample1,vdj_t_R1.fq.gz,vdj_t_R2.fq.gz,VDJ-T
vdj_sample1,vdj_b_R1.fq.gz,vdj_b_R2.fq.gz,VDJ-B

ThunderBio GEX+VDJ

Run thunderbio 5’-scRNA-seq together wit' single cell VDJ libraries:

starscope vdj_gex --conda \
                  --input sampleList.csv \
                  --genomeDir /path/to/STAR/reference/dir \

                  --trimLength 28 \
                  --soloCBstart 1 \
                  --soloCBlen 29 \
                  --soloUMIstart 30 \
                  --soloUMIlen 10 \

Main program

starscope uses starsolo an' Seurat package as th' core modules t' process
scRNA-seq data an' generate a concise html report. Th' workflow was
implemented by nextflow.

Runn'n pipeline:
starscope <run> --conda \
                --input sampleList.csv \
                --genomeDir /path/to/STAR/reference/dir \
                --genomeGTF /path/to/genomeGTF \
                --whitelist /path/to/whitelist \
                --trimLength 28 \
                --soloCBstart 1 \
                --soloCBlen 29 \
                --soloUMIstart 30 \
                --soloUMIlen 10 \
                -bg

Example input list (csv):
sample,fastq_1,fastq_2
sampleName,read1.fq.gz,/absolute/path/to/read2.fq.gz

Mak'n Reference:
starscope <mkref> --conda \
                  --genomeFasta /path/to/genome/fasta \
                  --genomeGTF /path/to/genome/gtf \
                  --refoutDir reference_out_dir \
                  -bg

starscope has four valid subcommands:
    run:            run th' scRNAseq analysis pipeline wit' nextflow
    mkref:          prepare STAR reference
    vdj_gex:        perform vdj+gex analysis
    check_version:  check software versions

Please use -h opt'n follow'n each subcommand t' get detail
o' th' opt'ns: e.g. starscope run -h

run command

Basic Usage:
============
starscope <run> --conda \
                --input sampleList.csv \
                --genomeDir /path/to/STAR/reference \
                --genomeGTF /path/to/genomeGTF \
                --whitelist /path/to/whitelist \
                --trimLength 28 \
                --soloCBstart 1 \
                --soloCBlen 29 \
                --soloUMIstart 30 \
                --soloUMIlen 10 \
                -bg

opt'ns:
  --conda               Use conda env t' run (true)
  --docker              Use docker container t' run
  --input               Input sample list, csv format, required
                        columns include "sample" fer sampleName,
                        "fastq_1" fer read1, "fastq_2" fer read2,
                        an' read1 be assumed t' contain cell
                        barcode.
  --genomeDir           Path o' STAR reference directory
  --genomeGTF           Path o' reference genome GTF file
  --whitelist           Path o' whitelist o' barcodes
  --trimLength          Min read length retained after cutadapt
                        trimm'n (28)
  --soloCBstart         Cell barcode start base 'n read1 (1)
  --soloCBlen           Cell barcode length (29)
  --soloUMIstart        UMI start base 'n read1 (30)
  --soloUMIlen          UMI length (10)
  --soloFeatures        Define whether only count UMI 'n exon region:
                        Gene or GeneFull which includes both exon an'
                        intron reads (GeneFull)
  --soloMultiMappers    Count'n method fer reads mapp'n t' multiple genes:
                        Unique or EM (Unique)
  --soloStrand          Library strandness, Forward fer thunderbio
                        3' RNA-seq, Reverse fer 5' RNA-seq (Forward)
  --config              Provide a custom nextflow config file t'
                        define all parameters
  --executor            Define executor o' nextflow (local), see:
                        https://www.nextflow.io/docs/latest/executor.html
  --cpus                CPUs t' use fer all processes (8)
  --mem                 Memory t' use fer all processes, please note
                        th' special format (32.GB)
  --noDepCheck          Do not check Java an' nextflow before
                        runn'n (false)
  -bg                   Runn'n th' pipeline 'n background (false)

Advance Usage

User could use a local config file an' submit jobs t' job scheduler on HPC (e.g. slurm):

starscope run --input sampleList.csv --config local.config -bg

Example config fer thunderbio 3’ scRNA-seq data:

params {
  genomeDir = "/path/to/STAR/reference/"
  genomeGTF = "/path/to/reference/genes.gtf"
  whitelist = "/path/to/whitelist"
  trimLength = 50
  soloCBstart = 1
  soloCBlen = 29
  soloUMIstart = 30
  soloUMIlen = 10
  soloMultiMappers = "Unique"
  enable_conda = true
}

process {
  //executor = "slurm" // uncomment this line t' submit jobs t' slurm job scheduler
  conda = "/path/to/miniconda3/envs/starscope_env"
  // adjust resources here
  withLabel: process_high {
    cpus = 8
    memory = 32.GB
  }
  withLabel: process_medium {
    cpus = 4
    memory = 20.GB
  }
  withLabel: process_low {
    cpus = 4
    memory = 20.GB
  }
  withName: REPORT {
    // adjust report memory config accord'n t' dataset size
    memory = 40.GB
  }
}

For docker container, please use th' config below:

params {
  genomeDir = "/path/to/STAR/reference/"
  genomeGTF = "/path/to/reference/genes.gtf"
  whitelist = "/path/to/whitelist"

  soloCBlen = 29

  soloMultiMappers = "Unique"

process {

  container = "registry-intl.cn-hangzhou.aliyuncs.com/thunderbio/thunderbio_scrnaseq_env:2.7.10a"
  withLabel: process_high {
    cpus = 8
    memory = 32.GB
  }
  withLabel: process_medium {
    cpus = 4
    memory = 20.GB
  }
  withLabel: process_low {
    cpus = 4
    memory = 20.GB
  }

All parameters could be found 'n th' starscope/scRNA-seq/nextflow.config file.

Output

  • results: main output directory

    • pipeline_info: store plain text execution_trace file
    • cutqc: store cutadapt trimm'n report generated by customized R markdown script
    • starsolo: store starsolo output files, includ'n barcode/UMI matrix, summary files etc.
    • qualimap: store qualimap report
    • report: contain main HTML report files

  • work: intermediate directory, could be safely removed after runn'n complete
